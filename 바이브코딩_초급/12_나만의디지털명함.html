FileReader API — 파일 미리보기

선택한 이미지 파일을 브라우저에서 바로 읽어 표시

FileReader API는 사용자가 선택한 파일을 서버 업로드 없이 브라우저에서 직접 읽어서 미리보기를 만드는 기술입니다. 로컬에서만 처리되므로 빠르고 안전합니다.
<!-- HTML --> <input type="file" accept="image/*" id="photoInput"> <img id="preview"> // JavaScript const input = document.getElementById('photoInput'); input.addEventListener('change', function(e) { const file = e.target.files[0]; // 선택된 파일 if (!file) return; const reader = new FileReader(); // 읽기 완료 시 실행될 함수 등록 reader.onload = function(e) { // e.target.result = base64 이미지 URL document.getElementById('preview').src = e.target.result; }; reader.readAsDataURL(file); // 읽기 시작 });
📋

FileReader 흐름

1️⃣ 사용자가 파일 선택
2️⃣ new FileReader() 생성
3️⃣ onload에 완료 콜백 등록
4️⃣ readAsDataURL()로 읽기
5️⃣ 결과(base64 URL)를 img.src에 설정

📸

html2canvas — HTML을 이미지로 저장

CDN으로 간단하게 사용할 수 있는 캡처 라이브러리

CDN 불러오기 + 사용법

HTML+JS <!-- CDN으로 라이브러리 불러오기 --> <script src="https://html2canvas.hertzen.com/dist/html2canvas.min.js" ></script> // 명함 캡처 후 저장 async function saveCard() { const card = document.getElementById('card'); // 캡처 (Promise 반환) const canvas = await html2canvas(card, { scale: 2, // 2배 고해상도 useCORS: true // 외부 이미지 허용 }); // 다운로드 링크 생성 const link = document.createElement('a'); link.download = '나의명함.png'; link.href = canvas.toDataURL(); link.click(); // 자동 다운로드 }

라이브러리란?

📦 라이브러리

다른 개발자가 미리 만들어둔 코드 묶음.
html2canvas는 HTML 요소를 이미지로 캡처하는 복잡한 기능을 단 두 줄로 사용 가능하게 해줍니다.

🌐 CDN (Content Delivery Network)

<script src="url">로 인터넷에서 직접 불러옵니다. 설치 없이 URL만 추가하면 바로 사용!

실시간 미리보기 패턴

입력창 변경 → 명함 즉시 업데이트

입력창마다 input 이벤트를 걸어두면 글자를 칠 때마다 명함이 실시간으로 업데이트됩니다. 이것이 이번 실습에서 배운 모든 개념의 종합 활용입니다.
// 모든 입력창에 이벤트 등록 ['name', 'title', 'company', 'email', 'phone', 'bio'] .forEach(fieldId => { document.getElementById(fieldId) .addEventListener('input', updatePreview); }); function updatePreview() { // 각 필드 값 읽어서 명함에 반영 const name = document .getElementById('name').value; document.getElementById('card-name') .textContent = name || '이름을 입력하세요'; // ... 다른 필드도 동일하게 }

자주 막히는 문제 & 해결법

여기서 멈추는 게 정상입니다 — 옆 사람도 같은 곳에서 막혀 있어요

🔍 "Go Live 버튼이 안 보여요"

VS Code 왼쪽 확장(Extensions) 메뉴에서 Live Server를 검색해 설치한 뒤, VS Code를 껐다가 다시 켜세요.

💾 "고쳤는데 화면이 그대로예요"

저장(Ctrl+S)이 먼저입니다! 저장하지 않으면 반영되지 않아요. 저장 후 브라우저에서 새로고침(F5) 하세요.

⏸️ "Cline이 멈춰서 아무것도 안 해요"

파일을 만들거나 고치기 전에 승인(Approve)을 기다리는 중입니다. Cline 채팅창의 승인 버튼을 눌러주세요.

🔤 "한글이 깨져 보여요"

AI에게 "한글이 깨져 보여. UTF-8 인코딩으로 고쳐줘"라고 요청하세요. VS Code 오른쪽 아래 인코딩 표시가 UTF-8인지도 확인!

🤖 만능 해결법: 오류 메시지나 이상한 화면을 그대로 복사(캡처)해서 AI에게 "이거 왜 안 돼? 고쳐줘"라고 붙여넣으세요. 대부분 해결됩니다.

코드 핵심 — 초기화 & 저장 흐름

localStorage로 마지막 입력을 기억하는 완성된 패턴

JS const FIELDS = ['name', 'title', 'company', 'email', 'phone', 'bio']; // 페이지 로드: 저장된 데이터 복원 function loadSaved() { const saved = JSON.parse(localStorage.getItem('cardData') || '{}'); FIELDS.forEach(f => { if (saved[f]) document.getElementById(f).value = saved[f]; }); updatePreview(); // 복원된 값으로 미리보기 } // 입력 시: 자동 저장 + 미리보기 업데이트 function onInput() { const data = {}; FIELDS.forEach(f => { data[f] = document.getElementById(f).value; }); localStorage.setItem('cardData', JSON.stringify(data)); updatePreview(); } FIELDS.forEach(f => document.getElementById(f).addEventListener('input', onInput)); loadSaved(); // 초기 실행
