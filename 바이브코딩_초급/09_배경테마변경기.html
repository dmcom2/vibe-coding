📌 CSS 변수

색상 값에 --main-color처럼 이름표를 붙여두는 것. 한 곳만 바꾸면 전체 적용!

문서 전체에 적용되는 기준점. CSS 변수를 여기 모아둡니다

📌 setProperty

JS에서 CSS 변수 값을 바꾸는 명령. 테마 전환의 핵심

📌 다크모드

📌 prefers-color-scheme

사용자 컴퓨터의 다크모드 설정을 읽어오는 방법

📌 localStorage

선택한 테마를 기억해두는 브라우저 저장소

CSS 변수 (Custom Properties)

색상을 변수로 정의해서 한 번에 바꾸는 방법

CSS 변수는 --변수명으로 정의하고 var(--변수명)으로 사용합니다.
JS로 변수 하나를 바꾸면 그 변수를 쓰는 모든 곳이 동시에 바뀝니다.
/* CSS - :root에 변수 선언 */ :root { --bg-color: #ffffff; --text-color: #1e293b; --accent: #7c3aed; --card-bg: #f8fafc; } /* 변수 사용 */ body { background: var(--bg-color); color: var(--text-color); } .card { background: var(--card-bg); border-color: var(--accent); }
💡

CSS 변수의 장점

색상이 100곳에 쓰여도 변수 하나만 바꾸면 끝!

기존 방식: style.color, style.backgroundColor... 일일이 바꿔야

변수 방식: setProperty 한 번으로 모두 변경

⚙️

setProperty — JS로 CSS 변수 변경

document.documentElement로 :root 변수에 접근

JS // :root = document.documentElement const root = document.documentElement; // CSS 변수 값 변경 root.style.setProperty('--bg-color', '#1e1e2e'); root.style.setProperty('--text-color', '#cdd6f4'); root.style.setProperty('--accent', '#cba6f7'); // 현재 변수 값 읽기 const current = getComputedStyle(root) .getPropertyValue('--bg-color'); // localStorage에 테마 저장 localStorage.setItem('theme', 'dark');

테마 객체 패턴

JS const THEMES = { light: { '--bg-color': '#ffffff', '--text-color': '#1e293b', '--accent': '#7c3aed' }, dark: { '--bg-color': '#1e1e2e', '--text-color': '#cdd6f4', '--accent': '#cba6f7' } }; function applyTheme(name) { const t = THEMES[name]; Object.entries(t).forEach(([k,v]) => root.style.setProperty(k, v) ); }
🌙

시스템 다크모드 자동 감지

사용자 OS 설정에 따라 자동으로 테마 적용

// 시스템 다크모드 여부 확인 const prefersDark = window .matchMedia('(prefers-color-scheme: dark)') .matches; if (prefersDark) { applyTheme('dark'); } else { applyTheme('light'); } // 시스템 테마 변경 감지 (실시간) window.matchMedia('(prefers-color-scheme: dark)') .addEventListener('change', e => { applyTheme(e.matches ? 'dark' : 'light'); });

HTML 파일 하나로 테마 변경 페이지를 만들어줘. CSS 변수(--bg-color, --text-color 등)를 사용해서 테마 전환 시 CSS 클래스만 바꾸면 전체가 변하도록 구현. 테마 종류 (6가지): 1. 라이트 (기본 흰 배경) 2. 다크 (어두운 배경) 3. 블루 (파란 계열) 4. 그린 (초록 계열) 5. 퍼플 (보라 계열) 6. 선셋 (주황-핑크 그라데이션) 페이지 구성: - 상단: 제목 + 현재 테마 이름 표시 - 가운데: 테마 선택 버튼들 (큰 컬러 원형 버튼) - 하단: 테마 미리보기 카드 (제목, 본문, 버튼 포함) 기능: - 선택된 테마 버튼에 체크 표시 - localStorage에 선택 기억 (새로고침해도 유지) - 테마 전환 시 부드러운 transition 효과 파일은 index.html 하나로, 코드에 한국어 주석 달아줘
👣 따라하기① 위 프롬프트 전체 복사▶② VS Code의 Cline 채팅창에 붙여넣고 전송▶③ Cline이 만든 파일 승인(Approve)▶④ 다음 슬라이드에서 실행
🎨 커스텀 색상사용자가 직접 배경색과 글자색을 컬러피커로 골라서 나만의 커스텀 테마를 만들 수 있게 해줘.
🌇 그라데이션 배경단색 배경 외에 두 가지 색상의 그라데이션 배경 테마도 추가하고 방향(각도)도 선택 가능하게 해줘.
💾 테마 저장마음에 드는 테마를 localStorage에 저장해서 즐겨찾기처럼 관리하고 이름을 붙일 수 있게 해줘.

자주 막히는 문제 & 해결법

여기서 멈추는 게 정상입니다 — 옆 사람도 같은 곳에서 막혀 있어요

🔍 "Go Live 버튼이 안 보여요"

VS Code 왼쪽 확장(Extensions) 메뉴에서 Live Server를 검색해 설치한 뒤, VS Code를 껐다가 다시 켜세요.

💾 "고쳤는데 화면이 그대로예요"

저장(Ctrl+S)이 먼저입니다! 저장하지 않으면 반영되지 않아요. 저장 후 브라우저에서 새로고침(F5) 하세요.

⏸️ "Cline이 멈춰서 아무것도 안 해요"

파일을 만들거나 고치기 전에 승인(Approve)을 기다리는 중입니다. Cline 채팅창의 승인 버튼을 눌러주세요.

🔤 "한글이 깨져 보여요"

AI에게 "한글이 깨져 보여. UTF-8 인코딩으로 고쳐줘"라고 요청하세요. VS Code 오른쪽 아래 인코딩 표시가 UTF-8인지도 확인!

🤖 만능 해결법: 오류 메시지나 이상한 화면을 그대로 복사(캡처)해서 AI에게 "이거 왜 안 돼? 고쳐줘"라고 붙여넣으세요. 대부분 해결됩니다.
🔍

코드 핵심 — 초기화 & 저장 패턴

페이지 로드 시 저장된 테마를 불러오는 패턴

JS // 페이지 로드 시 실행 function init() { // 1) 저장된 테마 확인 const saved = localStorage.getItem('theme'); if (saved && THEMES[saved]) { // 2a) 저장된 테마 적용 applyTheme(saved); markActive(saved); } else { // 2b) 시스템 설정 감지 const isDark = window.matchMedia( '(prefers-color-scheme: dark)' ).matches; applyTheme(isDark ? 'dark' : 'light'); } } // 테마 적용 + 저장 function applyTheme(name) { Object.entries(THEMES[name]).forEach(([k,v]) => document.documentElement.style.setProperty(k, v) ); localStorage.setItem('theme', name); markActive(name); } init(); // 로드 시 즉시 실행
