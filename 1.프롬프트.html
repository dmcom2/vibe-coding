1차시 · AI의 뇌 구조

AI는 어떻게 생각하나요?

🦜
인공지능 앵무새
AI의 작동 방식을 이해하는 핵심 비유

앵무새는 사람 말을 완벽하게 따라합니다. 하지만 의미를 이해하지는 않습니다.

LLM도 마찬가지입니다. 수천억 개의 문장을 학습해서 다음에 올 가장 확률 높은 단어를 예측할 뿐입니다.

💡
핵심: AI는 "정답을 아는 것"이 아니라 "그럴듯한 다음 단어를 이어 붙이는 것"입니다.
🔢 LLM (Large Language Model)의 원리
1
수천억 개 문장을 학습
2
단어들의 패턴과 관계를 통계로 저장
3
질문이 들어오면 가장 확률 높은 답을 생성
4
사람처럼 보이는 자연스러운 문장이 완성
⚠️
주의!
확률 기반이기 때문에 같은 질문에도 다른 답이 나올 수 있고, 틀린 내용을 자신있게 말할 수 있습니다.
1차시 · 맥락 좁히기

질문의 맥락이 결과를 바꾼다

🎯
AI는 바다처럼 넓은 가능성 중 하나를 고릅니다. 맥락이 좁을수록 원하는 곳에 정확히 도착합니다.
맥락이 없는 질문
# 이렇게 물으면... "보고서 써줘"
❌
어떤 보고서? 몇 페이지? 누구한테?
❌
형식은? 공식적? 비공식적?
❌
AI가 "보고서"에 대해 알고 있는 모든 형태를 아무거나 선택
😤
결과: 엉뚱하거나 너무 일반적인 내용
맥락이 있는 질문
# 이렇게 물으면... "우리 팀 3분기 마케팅 성과를 임원에게 보고하는 1페이지 요약 보고서를 써줘"
✅
대상: 임원 → 간결하고 핵심만
✅
분량: 1페이지 → 길지 않게
✅
내용: 마케팅 성과 → 관련 내용만
🎉
결과: 바로 쓸 수 있는 내용
📌

2차시 · CRTF 공식

CRTF — 4가지 요소 한눈에 보기

C
C
Context
맥락 · 상황
내가 처한 상황과 배경을 알려줍니다
예: "우리 회사는 중소기업이고..."
R
R
Role
역할 · 페르소나
AI에게 어떤 전문가로 행동할지 부여합니다
예: "마케팅 전문가로서..."
T
T
Task
작업 · 요청
무엇을 해달라고 정확히 지시합니다
예: "이메일 초안을 작성해줘"
F
F
Format
형식 · 출력 방식
결과물의 형태와 길이를 지정합니다
예: "3개 항목, 각 2줄 이내"
✨
4가지가 전부 없어도 됩니다. 있을수록 좋아질 뿐입니다. 가장 중요한 것은 T(Task)이고, 나머지는 품질을 높이는 재료입니다.
2차시 · CRTF 실전

CRTF 실전 적용 예시 비교

CRTF 없는 프롬프트
"제품 홍보 메일 써줘"
❌
어떤 제품인지 모름
❌
받는 사람이 누구인지 모름
❌
어떤 분위기의 메일인지 모름
❌
길이나 형식 지정 없음
결과: 일반적인 홍보 메일 템플릿 한 개 → 수정이 더 오래 걸림
CRTF 적용 프롬프트
# Context 우리 회사는 B2B SaaS 기업이고 이번 달 신규 기능을 출시했어 # Role 시니어 마케터 역할로 # Task 기존 고객에게 업그레이드를 유도하는 제품 업데이트 이메일을 작성해줘 # Format 제목 1줄, 본문 3문단(각 3줄 이내)
결과: 바로 발송 가능한 완성도 높은 이메일
📌
처음부터 CRTF 4가지를 다 쓰기 어려우면, T(Task)를 최대한 구체적으로 쓰는 것부터 시작하세요.
2차시 · Role 심화

역할(Role)을 주면 전문가가 된다

🎭
같은 질문이라도 역할을 어떻게 주느냐에 따라 답변의 깊이와 관점이 완전히 달라집니다.
역할 없음

고객에게 알려야 해?"

마케팅 전문가
"고객 심리 전문 마케터로서
가격 인상 안내 메일 전략을
알려줘"
고객 반응, 심리적 저항 최소화 방법을 구체적으로 제시
업계 특화 전문가
"B2B SaaS 업계 10년 경력
고객 성공 매니저 역할로
가격 인상 커뮤니케이션 해줘"

3차시 · 쪼개기 기술

큰 요청은 작게 쪼개라

🧩
AI에게 한 번에 너무 많은 것을 시키면 중간에 길을 잃거나 앞부분을 잊어버립니다. 코딩도, 문서 작업도 마찬가지입니다.
한 번에 다 요청
"직원 관리 시스템 만들어줘. 로그인, 회원가입, 직원 목록, 급여 관리, 휴가 신청, 성과 평가까지 다 넣어줘"
❌
AI가 어디서부터 시작해야 할지 혼란
❌
중반부터 초반 지시를 잊어버림
❌
완성도가 들쭉날쭉함
단계별 쪼개기
1
"로그인·회원가입 페이지만 만들어줘"
2
"직원 목록 CRUD 기능 추가해줘"
3
"급여 관리 탭 추가해줘"
4
"휴가 신청 기능 붙여줘"
각 단계 완료 후 확인 → 다음 단계로
💡
쪼개기 공식: 기능 1개 = 요청 1번. 완성을 확인하고 다음 단계로 넘어가세요.
3차시 · 울타리 치기

AI에게 절대 규칙을 심어라

🏡
.clinerules 파일은 AI가 매번 지켜야 하는 절대 규칙 파일입니다. 프로젝트 폴더에 한 번 만들어두면 AI가 항상 이 규칙을 따릅니다.
💡
파일 이름은 AI 도구마다 다릅니다. .clinerules는 Cline 전용 이름이고, Claude Code·Cursor·Gemini 등은 각자 다른 파일을 읽습니다. 아직 완전히 통일된 표준은 없지만, 최근 여러 도구가 함께 인식하기 시작한 공용 파일(AGENTS.md)이 등장하는 추세입니다.
Cline → .clinerules Claude Code → CLAUDE.md Cursor → .cursor/rules/*.mdc GitHub Copilot → .github/copilot-instructions.md Windsurf → .windsurfrules Gemini CLI → GEMINI.md 여러 도구 공용 시도 → AGENTS.md
실무 팁: 여러 도구를 함께 쓴다면 규칙을 한 파일에 기준으로 작성하고, 나머지 파일은 그 내용을 복사하거나 심볼릭 링크로 연결해 동기화하세요. AGENTS.md는 공식 국제 표준 기구가 정한 것은 아니고, 여러 AI 코딩 도구 진영이 자발적으로 맞춰가는 "사실상 표준(de facto)"에 가깝습니다.
왜 필요한가?
😤
규칙 없을 때
매번 "한국어로 써줘", "React 쓰지 마" 같은 말을 반복해야 함. AI가 종종 잊어버림.
😊
규칙 있을 때
한 번 설정하면 모든 대화에서 자동 적용. AI가 처음부터 규칙에 맞게 행동.
자주 쓰는 규칙 예시
# 언어 모든 응답은 한국어로 작성하세요. # 기술 스택 React가 아닌 순수 HTML/JS만 사용하세요. CSS 프레임워크 없이 인라인 스타일 금지. # 코드 스타일 함수는 최대 20줄을 넘지 마세요. 변수명은 한국어 주석으로 설명하세요. # 작업 방식 구현 전 반드시 계획을 먼저 설명하세요. 한 번에 한 기능씩만 수정하세요.
📌
만드는 법 (Cline 기준): 프로젝트 폴더에 .clinerules 파일 생성 → 규칙 작성 → 저장. 끝입니다. 다른 도구를 쓴다면 위 표에서 해당 파일 이름으로만 바꾸면 원리는 동일합니다.

④ 종합 전처리 요청 (원스톱)
# 복사한 엑셀 데이터 붙여넣고 "아래 데이터를 AI 분석에 적합하게 전처리해줘. 조건: 1. 병합 셀 해제 후 값 채우기 2. 빈 행/열 삭제 3. 날짜 → YYYY-MM-DD 통일 4. 금액 → 쉼표/기호 제거, 숫자만 5. 머리글 → 직관적으로 수정 6. 결과는 CSV 형식으로 출력 [데이터 붙여넣기]"
⑤ 전처리 후 바로 분석 이어서 요청
"위에서 정제한 데이터로 바로 월별 매출 추이를 분석하고 증감률과 특이사항을 요약해줘"
전처리 → 분석을 하나의 대화에서 연속으로 처리 가능
⚠️
주의: AI가 정제한 결과도 반드시 한 번은 눈으로 확인하세요. 원본 대비 행 수, 컬럼 수가 맞는지 체크!
3차시 · AI 밥상 차리기 ④

데이터 형식을 바꾸면 AI가 훨씬 잘 이해한다

🍱
엑셀에서 그냥 복붙하는 것보다 CSV·JSON·Markdown 형식으로 변환해서 주면 AI가 구조를 오해 없이 파악합니다. AI에게 변환을 시키면 됩니다. 직접 안 해도 됩니다.

4차시 · 환각 현상

AI가 거짓말을 하는 이유

👻
환각(Hallucination)이란?
AI가 존재하지 않는 사실을 자신있게 말하는 현상. 거짓말을 하는 게 아니라, 확률적으로 가장 "그럴듯한" 내용을 생성하다 보니 발생합니다.
가짜 출처
존재하지 않는 논문, 책, 기사를 있는 것처럼 인용합니다.
출처는 반드시 직접 확인!
날짜·숫자 오류
통계, 날짜, 수치를 틀리게 말하면서도 확신에 찬 어조로 말합니다.
수치는 원본 데이터 확인!
코드 환각
없는 함수, 라이브러리를 마치 있는 것처럼 코드를 작성합니다.
코드는 반드시 실행 테스트!
🔍 환각 방지 4원칙
1
중요한 사실은 다른 도구로 검증
2
AI에게 "출처를 알려줘"라고 추가 질문
3
"확실하지 않으면 모른다고 해" 지시
4
코드·법률·의료는 전문가 최종 확인
🛡️
올바른 자세
AI의 답변을 "초안"으로 받아들이고 중요한 정보는 검증하는 습관이 필요합니다.

💪
