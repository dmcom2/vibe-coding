시작은 나만의 엑셀 전문가 챗봇부터

GPTs / GEMs가 뭔가요?
ChatGPT의 GPTs, Gemini의 GEMs는 지침(시스템 프롬프트)을 미리 넣어둔 나만의 맞춤 챗봇입니다. 한 번 잘 만들어 두면 매번 역할·조건 설명 없이 바로 전문가와 일할 수 있습니다.
"엑셀 일잘러" 챗봇의 능력
· 수식·함수 생성 + 구버전(2016/2019) 대안 동시 제공
· 스프레드시트 분석·피벗 설계·차트 추천
· VBA 자동화 코드 생성 (에러 처리 포함)
· PDF→엑셀 변환, SQL 쿼리 생성까지
// 챗봇 지침의 핵심 뼈대 (실습 1에서 전문 제공)

미션: 이 GPT는 Excel 전 과정 도우미입니다.
원칙: "최소한의 질문 → 최대한의 실행물"

응답 형식 (항상 고정):
　1. 요약 (3~5줄)
　2. 즉시 복사 블록 (수식/코드/SQL)
　3. 구현 절차 (번호 목록)
　4. 검증 / 오류 대처
　5. (옵션) 성능·가독성 개선 팁

→ 응답 형식을 지침에 박아두는 것이 핵심!
　 매번 일정한 품질의 답을 받게 됩니다
왜 챗봇부터 만드나요? 1부~2부의 모든 실습(수식, VBA, GAS)을 이 챗봇에게 시키게 됩니다. 좋은 지침을 가진 챗봇은 모든 자동화 작업의 품질을 한 단계 끌어올리는 기반 공사입니다.

"이 작업을 위한 최적의 프롬프트를 먼저 만들어줘"라고 요청 → AI가 Role·Context·Task·Output 구조·Constraints를 갖춘 전문가급 프롬프트를 생성 → 그 프롬프트로 코드 생성.
메타 프롬프팅 작업 흐름
① 하고 싶은 일을 대충 설명
② "최적 프롬프트 만들어줘"
③ AI가 전문가 프롬프트 생성
④ 내가 검토·수정
⑤ 그 프롬프트로 실행

상황

1단계 — 개발정의서 (원본 프롬프트 원문)
HR 업무지원 챗봇을 외부 클라우드(Firebase/Firestore) 없이 로컬 PC에서 RAG 챗봇으로 구현하려고 한다. 로컬 PC 호스팅형으로 한 대의 PC 또는 노트북에서 웹앱과 백엔드를 함께 실행하는 방식이다. 파일럿, PoC, 부서 단위 테스트에 사용하려고 한다. 제한사항: 사내 문서와 정보의 외부 저장은 금지 - 문서는 회사 내부에만 저장 - 인덱스도 내부 저장 - 웹앱도 내부망에서만 운영 - 단, AI API 호출은 외부 인터넷으로 허용 미션: 로컬 PC PoC, 한 대의 PC에서 먼저 검증 - 문서 폴더 준비, 문서 20~50개만 선택 - 질문/답변 화면 확인 - 답변에 출처 붙이기 확인 1단계로 **개발정의서** 먼저 만들어줘 (프롬프트, 코드는 주지 말고)
2~5단계 — 체인 이어가기 (원본 프롬프트 원문)
[2단계] 이어서 '화면(UI/UX) 디자인 정의서' 제안해줘 (프롬프트, 코드 등 만들지 마) [3단계] '개발정의서', '화면 정의서'에 이어서 회사 보안 지침(API Key 환경변수 관리, 기밀정보 입력 금지, 최소 권한 원칙, 코드 리뷰 후 배포)을 반영한 '개발 명세서' 만들어줘 [4단계] 지금까지의 문서들을 토대로 '기술 스택 및 개발 환경 정의서'와 '프로토타입용 코드 작성 프롬프트'를 만들어주고, 모든 내용을 총정리한 'Master Plan.md'도 만들어줘 [5단계] 위 문서들을 토대로 implementation plan을 생성하고, 프로토타입을 코딩해줘. 순차적으로 검증하면서 구현할 수 있는 프로토타입 가이드도 함께 줘.

실습 8 · Dify로 기술규격 챗봇 만들기 (노코드)
난이도 ★★☆ · 약 30분 · 4부 RAG 챗봇 · 최종 실습
상황

구축 절차 (원본 가이드 요약)
① Dify Cloud 가입 → 워크스페이스 생성 (언어: Korean) ② 설정 → Model Provider → OpenAI(또는 Anthropic) API Key 등록 ③ Knowledge → Create Knowledge → 문서 업로드 - 청크 설정: Chunk Size 500~1000자 / Overlap 50~100자 - 표·도면 많은 문서는 크게(1000~1500자), Q&A 문서는 작게(300~500자) ④ Studio → Create Application → "Chat App" 선택 ⑤ Context에 지식베이스 연결 - Retrieval: Semantic Search / Top K: 3~5 / Score Threshold: 0.5~0.7 ⑥ Model: gpt-4o-mini (경제적) / Temperature: 0.3 (정확성 우선) ⑦ Instructions에 아래 시스템 프롬프트 입력 ⑧ 우측 미리보기로 테스트 → Publish로 배포 (웹 링크 공유)
시스템 프롬프트 (원본 실습 원문)
당신은 우리 회사의 전문 기술 지원 AI 어시스턴트입니다. 역할: - 제품 사양서, 매뉴얼, 도면 등의 기술 문서를 기반으로 정확한 정보 제공 - 엔지니어, 영업팀, 고객의 기술 문의에 친절하고 명확하게 답변 - 불확실한 정보는 추측하지 않고, 관련 문서를 확인하도록 안내 답변 원칙: 1. 지식베이스에서 찾은 정보만 답변 (임의 추측 금지) 2. 출처 항상 명시 (문서명, 페이지/섹션) 3. 기술 용어는 정확히 사용하되, 필요시 쉽게 부연 설명 4. 수치 데이터는 단위와 함께 정확히 표기 5. 정보가 없으면 "해당 정보가 제공된 문서에서 확인되지 않습니다"라고 솔직히 답변 답변 형식: - 간결한 요약 먼저, 상세 정보는 필요시 제공 - 표나 목록 형태로 정리하여 가독성 향상 금지 사항: - 확인되지 않은 정보를 마치 사실인 것처럼 답변 - 가격, 재고, 납기 등 실시간 정보 제공 (해당 부서 문의 안내)
