1차시 · AI의 뇌 구조

AI는 어떻게 생각하나요?

🦜
인공지능 앵무새
AI의 작동 방식을 이해하는 핵심 비유

앵무새는 사람 말을 완벽하게 따라합니다. 하지만 의미를 이해하지는 않습니다.

LLM도 마찬가지입니다. 수천억 개의 문장을 학습해서 다음에 올 가장 확률 높은 단어를 예측할 뿐입니다.

💡
핵심: AI는 "정답을 아는 것"이 아니라 "그럴듯한 다음 단어를 이어 붙이는 것"입니다.
🔢 LLM (Large Language Model)의 원리
1
수천억 개 문장을 학습
2
단어들의 패턴과 관계를 통계로 저장
3
질문이 들어오면 가장 확률 높은 답을 생성
4
사람처럼 보이는 자연스러운 문장이 완성
⚠️
주의!
확률 기반이기 때문에 같은 질문에도 다른 답이 나올 수 있고, 틀린 내용을 자신있게 말할 수 있습니다.
1차시 · 맥락 좁히기

질문의 맥락이 결과를 바꾼다

🎯
AI는 바다처럼 넓은 가능성 중 하나를 고릅니다. 맥락이 좁을수록 원하는 곳에 정확히 도착합니다.
맥락이 없는 질문
# 이렇게 물으면... "보고서 써줘"
❌
어떤 보고서? 몇 페이지? 누구한테?
❌
형식은? 공식적? 비공식적?
❌
AI가 "보고서"에 대해 알고 있는 모든 형태를 아무거나 선택
😤
결과: 엉뚱하거나 너무 일반적인 내용
맥락이 있는 질문
# 이렇게 물으면... "우리 팀 3분기 마케팅 성과를 임원에게 보고하는 1페이지 요약 보고서를 써줘"
✅
대상: 임원 → 간결하고 핵심만
✅
분량: 1페이지 → 길지 않게
✅
내용: 마케팅 성과 → 관련 내용만
🎉
결과: 바로 쓸 수 있는 내용
📌
1차시 핵심: AI에게 맥락을 줄수록 원하는 결과에 가까워집니다. 다음 2차시에서 이것을 공식화합니다.

2차시 · CRTF 공식

CRTF — 4가지 요소 한눈에 보기

C
C
Context
맥락 · 상황
내가 처한 상황과 배경을 알려줍니다
예: "우리 회사는 중소기업이고..."
R
R
Role
역할 · 페르소나
AI에게 어떤 전문가로 행동할지 부여합니다
예: "마케팅 전문가로서..."
T
T
Task
작업 · 요청
무엇을 해달라고 정확히 지시합니다
예: "이메일 초안을 작성해줘"
F
F
Format
형식 · 출력 방식
결과물의 형태와 길이를 지정합니다
예: "3개 항목, 각 2줄 이내"
✨
4가지가 전부 없어도 됩니다. 있을수록 좋아질 뿐입니다. 가장 중요한 것은 T(Task)이고, 나머지는 품질을 높이는 재료입니다.
2차시 · CRTF 실전

CRTF 실전 적용 예시 비교

CRTF 없는 프롬프트
"제품 홍보 메일 써줘"
❌
어떤 제품인지 모름
❌
받는 사람이 누구인지 모름
❌
어떤 분위기의 메일인지 모름
❌
길이나 형식 지정 없음
결과: 일반적인 홍보 메일 템플릿 한 개 → 수정이 더 오래 걸림
CRTF 적용 프롬프트
# Context 우리 회사는 B2B SaaS 기업이고 이번 달 신규 기능을 출시했어 # Role 시니어 마케터 역할로 # Task 기존 고객에게 업그레이드를 유도하는 제품 업데이트 이메일을 작성해줘 # Format 제목 1줄, 본문 3문단(각 3줄 이내)
결과: 바로 발송 가능한 완성도 높은 이메일
📌
처음부터 CRTF 4가지를 다 쓰기 어려우면, T(Task)를 최대한 구체적으로 쓰는 것부터 시작하세요.
2차시 · Role 심화

역할(Role)을 주면 전문가가 된다

🎭
같은 질문이라도 역할을 어떻게 주느냐에 따라 답변의 깊이와 관점이 완전히 달라집니다.
역할 없음
"가격 인상을 어떻게
고객에게 알려야 해?"
일반적인 정보를 나열. 구체성이 떨어짐.
마케팅 전문가
"고객 심리 전문 마케터로서
가격 인상 안내 메일 전략을
알려줘"
고객 반응, 심리적 저항 최소화 방법을 구체적으로 제시
업계 특화 전문가
"B2B SaaS 업계 10년 경력
고객 성공 매니저 역할로
가격 인상 커뮤니케이션 해줘"
업계 특성에 맞는 전략, 이탈 방지 방법까지 조언
👍 좋은 역할 부여 방법
✓
직책 + 경력 + 전문 분야 조합
✓
"당신은 ~입니다" 또는 "~로서"
✓
내가 필요한 관점에 맞는 역할
💡 업무별 추천 역할
문서
전문 카피라이터, 편집장
분석
데이터 분석가, 컨설턴트
코딩
시니어 개발자, 풀스택 엔지니어

3차시 · 쪼개기 기술

큰 요청은 작게 쪼개라

🧩
AI에게 한 번에 너무 많은 것을 시키면 중간에 길을 잃거나 앞부분을 잊어버립니다. 코딩도, 문서 작업도 마찬가지입니다.
한 번에 다 요청
"직원 관리 시스템 만들어줘. 로그인, 회원가입, 직원 목록, 급여 관리, 휴가 신청, 성과 평가까지 다 넣어줘"
❌
AI가 어디서부터 시작해야 할지 혼란
❌
중반부터 초반 지시를 잊어버림
❌
완성도가 들쭉날쭉함
단계별 쪼개기
1
"로그인·회원가입 페이지만 만들어줘"
2
"직원 목록 CRUD 기능 추가해줘"
3
"급여 관리 탭 추가해줘"
4
"휴가 신청 기능 붙여줘"
각 단계 완료 후 확인 → 다음 단계로
💡
쪼개기 공식: 기능 1개 = 요청 1번. 완성을 확인하고 다음 단계로 넘어가세요.
3차시 · 울타리 치기

AI에게 절대 규칙을 심어라

🏡
.clinerules 파일은 AI가 매번 지켜야 하는 절대 규칙 파일입니다. 프로젝트 폴더에 한 번 만들어두면 AI가 항상 이 규칙을 따릅니다.
왜 필요한가?
😤
규칙 없을 때
매번 "한국어로 써줘", "React 쓰지 마" 같은 말을 반복해야 함. AI가 종종 잊어버림.
😊
규칙 있을 때
한 번 설정하면 모든 대화에서 자동 적용. AI가 처음부터 규칙에 맞게 행동.
자주 쓰는 규칙 예시
# 언어 모든 응답은 한국어로 작성하세요. # 기술 스택 React가 아닌 순수 HTML/JS만 사용하세요. CSS 프레임워크 없이 인라인 스타일 금지. # 코드 스타일 함수는 최대 20줄을 넘지 마세요. 변수명은 한국어 주석으로 설명하세요. # 작업 방식 구현 전 반드시 계획을 먼저 설명하세요. 한 번에 한 기능씩만 수정하세요.
📌
만드는 법: 프로젝트 폴더에 .clinerules 파일 생성 → 규칙 작성 → 저장. 끝입니다.

4차시 · 환각 현상

AI가 거짓말을 하는 이유

👻
환각(Hallucination)이란?
AI가 존재하지 않는 사실을 자신있게 말하는 현상. 거짓말을 하는 게 아니라, 확률적으로 가장 "그럴듯한" 내용을 생성하다 보니 발생합니다.
가짜 출처
존재하지 않는 논문, 책, 기사를 있는 것처럼 인용합니다.
출처는 반드시 직접 확인!
날짜·숫자 오류
통계, 날짜, 수치를 틀리게 말하면서도 확신에 찬 어조로 말합니다.
수치는 원본 데이터 확인!
코드 환각
없는 함수, 라이브러리를 마치 있는 것처럼 코드를 작성합니다.
코드는 반드시 실행 테스트!
🔍 환각 방지 4원칙
1
중요한 사실은 다른 도구로 검증
2
AI에게 "출처를 알려줘"라고 추가 질문
3
"확실하지 않으면 모른다고 해" 지시
4
코드·법률·의료는 전문가 최종 확인
🛡️
올바른 자세
AI의 답변을 "초안"으로 받아들이고 중요한 정보는 검증하는 습관이 필요합니다.
4차시 · New Task 전략

꼬이면 새 판을 깔아라

🔄
AI가 같은 오류를 반복한다면, 대화가 길어질수록 AI가 초반 맥락을 잊고 오염됩니다. 새 대화를 시작하세요.
🚫 New Task를 눌러야 할 신호
😤
같은 에러가 3번 이상 반복될 때
😤
AI의 답변이 점점 엉뚱해질 때
😤
대화 내용이 50개 이상 쌓였을 때
😤
방향이 완전히 바뀌는 작업을 시작할 때
✅ 효율적인 New Task 방법
1
이전 작업 결과 파일 저장 후
2
새 대화창(+ New Task) 열기
3
이전 결과와 새 목표를 함께 붙여넣기
4
깨끗한 뇌 상태에서 다시 시작
💡
새 판 = 포기가 아닌 리셋 전략
📌
기억하세요: AI와 싸우지 마세요. 길이 막히면 다른 길로 가는 것이 훨씬 빠릅니다.
4차시 · 에러 핸들링

에러를 두려워하지 마라

💪
에러 메시지는 AI가 다음에 무엇을 해야 하는지 알려주는 힌트입니다. 해석하려 하지 말고 그대로 복사해서 넘겨주세요.
❌ 잘못된 에러 처리 방식
직접 해석하려는 것
"TypeError: undefined is not a function이 뭐야?" → 혼자 구글링 시간 낭비
에러 일부만 복사하는 것
첫 줄만 복사 → AI가 전체 맥락을 모름
막연하게 "안 돼요"
"버튼이 안 눌려요" → AI가 원인을 추측해야 함
✅ 올바른 에러 처리 루틴
1
에러 메시지 전체 복사 (Ctrl+A)
2
"이 에러 고쳐줘:
[에러 내용 붙여넣기]"
3
AI가 제안한 해결책 적용
4
결과 확인 → 해결 안 되면 2번 반복
5
3번 이상 반복 시 → New Task
🏆
핵심 마인드셋: 에러는 실패가 아닙니다. AI가 올바른 방향을 찾기 위한 과정입니다. 평균 3~5번 반복이 정상입니다.

배우기 전 vs 배운 후

교육 전 — 흔한 실수들
😤
"AI가 멍청해서 제대로 못 하는구나..."
😤
"써줘" 한 마디로 요청 → 엉뚱한 결과
😤
한 번에 열 가지 요청 → 뒤죽박죽
😤
에러가 나면 패닉 → 포기
😤
AI 답변을 그대로 믿어 사고 발생
교육 후 — 달라진 모습
😊
"AI의 특성을 알고 맞게 활용해야지"
😊
CRTF 4요소로 구체적 지시 → 높은 품질
😊
기능 1개씩 쪼개서 요청 → 완성도 UP
😊
에러 복사 → AI에게 전달 → 해결
😊
중요 정보는 검증 → 안전한 활용
🚀
같은 AI, 다른 결과

실천 과제 · 오늘부터 바로

오늘 바로 실천하는 3가지

01
지금 하고 있는 업무에 CRTF 적용해보기
이메일 작성, 보고서 초안, 회의 안건 정리 등 오늘 해야 하는 업무 하나에 C+R+T+F를 넣어서 AI에게 요청해보세요. 차이를 바로 느낄 수 있습니다.
